The event is given by the L2CAP layer when a connection update request is received from the slave. The upper layer which receives this event has to respond by sending a aci_l2cap_connection_parameter_update_resp command.

Parameters
Connection_HandleHandle of the connection related to this L2CAP procedure.
IdentifierThis is the identifier which associates the request to the response.
L2CAP_LengthLength of the L2CAP connection update request.
Interval_MinMinimum value for the connection event interval. This shall be less than or equal to Conn_Interval_Max. Time = N * 1.25 msec. Values:
  • 0x0006 (7.50 ms) ... 0x0C80 (4000.00 ms)
Interval_MaxMaximum value for the connection event interval. This shall be greater than or equal to Conn_Interval_Min. Time = N * 1.25 msec. Values:
  • 0x0006 (7.50 ms) ... 0x0C80 (4000.00 ms)
Slave_LatencyPeripheral latency for the connection in number of connection events. Values:
  • 0x0000 ... 0x01F3
Timeout_MultiplierDefines connection timeout parameter in the following manner: Timeout Multiplier * 10ms. Values:
  • 10 (100 ms) ... 3200 (32000 ms)
Return values
None

Event generated when a request or a response is received from the peer to create one L2CAP Credit Based Flow Control channel or one or more (up to 5) L2CAP Enhanced Credit Based Flow Control channels.

Parameters
Connection_HandleConnection handle that identifies the connection. Values:
  • 0x0000 ... 0x0EFF
Event_TypeValues:
  • 0x00: L2CAP_CFC_CONN_REQ
  • 0x01: L2CAP_CFC_CONN_RESP
  • 0x02: L2CAP_ECFC_CONN_REQ
  • 0x03: L2CAP_ECFC_CONN_RESP
IdentifierThis is the identifier which associates the request to the response.
SPSMSimplified Protocol/Service Multiplexer Values:
  • 0x0001 ... 0x00FF
Peer_MTUThe maximum SDU size (in octets) that the remote L2CAP layer entity can receive on this channel. Values:
  • 64 ... 65535
Peer_MPSThe maximum PDU payload size (in octets) that the remote L2CAP layer entity can receive on this channel. Values:
  • 64 ... 65535
Initial_CreditsThe number of K-frames that can be sent to the L2CAP layer entity of the peer device as soon as the L2CAP channel is established.
ResultResult of the connection request. Values:
  • 0x0000: L2CAP_CONN_SUCCESSFUL
  • 0x0002: L2CAP_CONN_FAIL_SPSM_NOT_SUPPORTED
  • 0x0004: L2CAP_CONN_FAIL_INSUFFICIENT_RESOURCES
  • 0x0005: L2CAP_CONN_FAIL_INSUFFICIENT_AUTHENTICATION
  • 0x0006: L2CAP_CONN_FAIL_INSUFFICIENT_AUTHORIZATION
  • 0x0007: L2CAP_CONN_FAIL_KEY_SIZE_TOO_SHORT
  • 0x0008: L2CAP_CONN_FAIL_INSUFFICIENT_ENCRYPTION
  • 0x0009: L2CAP_CONN_FAIL_INVALID_SOURCE_CID
  • 0x000A: L2CAP_CONN_FAIL_SOURCE_CID_ALREADY_ALLOCATED
  • 0x000B: L2CAP_CONN_FAIL_UNACCEPTABLE_PARAMETERS
  • 0x000C: L2CAP_CONN_FAIL_INVALID_PARAMETERS
  • 0x000D: L2CAP_CONN_FAIL_NO_FURTHER_INFORMATON
  • 0x000E: L2CAP_CONN_FAIL_AUTHENTICATION_PENDING
  • 0x000F: L2CAP_CONN_FAIL_AUTHORIZATION_PENDING
CID_CountNumber of channels to be created which are present in the L2CAP Credit Based Connection Request. The effective number of channels that will be opened can be less than this value (see Local_CID) . Values:
  • 0x01 ... 0x05

Parameters
Connection_HandleConnection handle that identifies the connection. Values:
  • 0x0000 ... 0x0EFF
Event_TypeValues:
  • 0x00: L2CAP_ECFC_RECONFIGURE_REQ
  • 0x01: L2CAP_ECFC_RECONFIGURE_RESP
ResultResult of the connection request if Event_Type is L2CAP_ECFC_CONN_RESP. To be ignored if Event_Type is L2CAP_ECFC_CONN_REQ. Values:
  • 0x0000: L2CAP_RECONFIG_SUCCESSFUL
  • 0x0001: L2CAP_RECONFIG_FAIL_MTU_REDUCTION_NOT_ALLOWED
  • 0x0002: L2CAP_RECONFIG_FAIL_MPS_REDUCTION_NOT_ALLOWED
  • 0x0003: L2CAP_RECONFIG_FAIL_INVALID_CID
  • 0x0004: L2CAP_RECONFIG_FAIL_UNACCEPTABLE_PARAMETERS
IdentifierThis is the identifier which associates the request to the response.
Peer_MTUThe maximum SDU size (in octets) that the remote L2CAP layer entity can receive on this channel after successful reconfiguration. Values:
  • 64 ... 65535
Peer_MPSThe maximum PDU payload size (in octets) that the remote L2CAP layer entity can receive on this channel after successful reconfiguration. Values:
  • 64 ... 65535
CID_CountNumber of Channels that are going to be created. Values:
  • 0x01 ... 0x05

Event raised when an L2CAP_FLOW_CONTROL_CREDIT_IND is received from the peer, which means that it is capable of receiving additional K-frames (for example after it has processed one or more K-frames) in LE Credit Based Flow Control.

Parameters

CIDThe local channel endpoint that identifies the L2CAP channel.
TX_CreditsThe number of additional k-frames that the peer's L2CAP layer entity can receive.
TX_Credit_BalanceRemaining number of K-frames that local L2CAP layer entity can currently send to the peer. Values:
  • 1 ... 65535
