BlueNRG-X Radio Init Wizard

The BleNRG-X Radio Init Wizard is a PC application which allows to define the proper values required for the correct Bluetooth LE stack radio initialization, based on the specific user application scenario. As consequence of the user choices, a configuration header file (*_config.h) is generated: this file must be used on the user demonstration application folder. This document is valid for BlueNRG-LP/LPS Bluetooth LE stack v3.x.

How to Run

  • User can run this PC application by clicking on the BlueNRG-X Radio Init Wizard icon under: Start menu folder, ST BlueNRG-LP_LPS DK X.X.X, BlueNRG-X Radio Init Wizard.
Figure 1: BlueNRG-X Radio Init Wizard

Main user interface window

  • On BlueNRG-X Radio Init Wizard main window user can select:
    • Device type (BlueNRG-LP/LPS)
    • Bluetooth LE stack version (3.0, 3.1, 3.1a/3.1b, 3.2, 3.2a)
  • In the left section of the BlueNRG-X Radio Init Wizard Utility main window, user can select the following topics allowing to define the specific radio initialization parameters based on the specific Bluetooth LE application requirements:
    1. Stack configuration
    2. Radio Configuration
    3. Service Configuration
    4. Connection Configuration
    5. Overview
    6. Output

Stack configuration

  • It allows to define some stack configuration options:
    1. Device name characteristic
    2. Select which privacy level is used on user application (disabled, privacy host, privacy controller).
    3. LE secure connections (enabled/disabled)
    4. Master GAP role (enabled/disabled)
    5. Data length extension (enabled/disabled)
    6. LE 2M and LE Coded PHY (enabled/disabled)
    7. Extended Advertising/scanning (enabled/disabled)
    8. L2CAP Connection-Oriented Channels (enabled/disabled)
    9. New Bluetooth Low Energy v3.1 or later modular configurations options:
      • Periodic advertising and PAST (enabled/disabled)
      • Constant Tone Extension (enabled/disabled; not supported on BlueNRG-LP).
      • LE Power Control & Path Loss Monitoring (enabled/disabled)
      • Connection features support
    10. New Bluetooth Low Energy v3.2 or later modular configurations options:
      • LE channel classification (enabled/disabled)
      • Broadcast Isochronous streams (enabled/disabled)
      • EATT support (enabled/disabled)
      • Connection Isochronous streams (enabled/disabled)
      • Connection subrating (enabled/disabled)

Radio configuration

  • It allows to define the radio configuration parameters:
    1. Maximum number of simultaneous radio tasks that the device can support:
      • A radio task is an internal link layer state machine which handles a specific radio activity (connection, advertising, scanning).
    2. Optional additional number of memory blocks to be added to the Bluetooth LE stack packet manager
    3. Max Attribute MTU
    4. Number of Advertising Data sets
    5. Number of Auxiliary Scan Slots
    6. White list size
    7. Number of concurrent GATT Client procedures
    8. L2CAP max payload size
    9. L2CAP max number channels for credit flow control
    10. New parameters supported on Bluetooth LE stack v3.1 or later:
      • Number of slots for synchronizing
      • Number of Antenna IDs used in CTE connection (where applicable)
      • Number of IQ samples used in CTE connection (where applicable)
      • Size of the internal FIFO used for critical controller events produced by the ISR (e.g. rx data packets). 256 is the typical defualt value.
      • Size of the internal FIFO used for non-critical controller events produced by the ISR (e.g. advertising or IQ sampling reports).768 is the typical defualt value.
      • Size of the internal FIFO used for controller and host events produced outside the ISR. 1024 is the typical defualt value.
    11. New parameters supported on Bluetooth LE stack v3.2 or later:
      • Number of EATT channels
  • NOTEs:
    1. Refer to the Bluetooth LE Stack Memory Management documentation available on BlueNRG-LP/LPS DK SW package for parameters description and usage.

Service configuration

  • It allows to define the services and characteristics configuration in terms of:
    1. Number of used services;
    2. For each service, it allows to define the number of characteristics and the related features (UUID, properties) impacting the GATT database size (1).
  • NOTEs:
    1. Refer to the Bluetooth LE GATT database size documentation available on BlueNRG-LP/LPS DK SW package for parameters description and usage.

Connection configuration

  • It allows to define the connection configuration parameters:
    1. Max connection event length
    2. Low Speed Crystal
    3. Sleep Clock Accuracy in ppm
    4. High Speed Startup Time in us
  • NOTEs:
    1. Refer to the Bluetooth Low Energy specification for parameters description.

Overwiev

  • It allows to get an overview of the overall defined values for the device configuration parameters based on the selected user application scenario.
  • On GATT/GAP Database section, user can also get a estimation about the amount of memory, in bytes, needed for overall storage of GATT database elements and other data structures whose size depends on several parameters (i.e: maximum number of simultaneous radio tasks that the device can support, ..). This could be useful for allowing user to better tune the overall application configuration options in order to fit within the overall available RAM memory resources.

Output window

  • It allows to get an overview of the application configuration header file which contains all the required Bluetooth LE stack radio initialization parameters values.
  • This file can be saved with the selected "<user_name>" application. A file "<user_name>_config.h" is generated. User can then copy it on its application project folder and include it on his main application.
